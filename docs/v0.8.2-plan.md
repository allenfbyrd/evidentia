## What shipped

### Phase 1 ŌĆö Review-deferral closure

- **F-V81-S1** ŌĆö `evidentia mcp serve --allow-root <path>` flag
  gates file-path tool inputs (`gap_analyze.inventory_path`,
  `gap_diff.{base,head}_report_path`) via
  `evidentia_core.security.paths.validate_within`. Out-of-root
  paths surface as `PathTraversalError` (a `ValueError` subclass).
  Default (no flag) preserves v0.8.1 stdio behavior; non-loopback
  HTTP/SSE without the flag emits an additional startup warning.
- **F-V81-S2** ŌĆö AuthProvider construction moved from import-
  time module-level ŌåÆ FastAPI `lifespan` async context manager.
  Importing `evidentia_api.app` is now side-effect-free; env var
  `EVIDENTIA_API_AUTH_TOKEN_FILE` is read at app startup.
  Explicit injection via `create_app(auth_provider=...)` continues
  to take precedence.

### Phase 2 ŌĆö Supply-chain G4 (foundation; activation deferred)

- `docker/requirements.txt` regenerated against the v0.8.2 dep
  tree with SHA256 hashes per transitive (~140 deps). Generated
  by `pip-compile --generate-hashes` inside the pinned Linux
  base image (uvloop is Linux-only; pip-compile on Windows hosts
  misses it).
- `scripts/bump_version.py --regenerate-requirements` wraps
  `pip-compile` invocation atomically with version bump.
- Activation of `pip install --require-hashes` Dockerfile flip
  **deferred to v0.8.3** per ┬¦25.6 R1: release.yml's `uv build`
  is not byte-identical across hosts (no SOURCE_DATE_EPOCH
  yet), so SHA256 hashes computed pre-tag don't match what
  release.yml uploads to PyPI. v0.8.3 closes via either
  reproducible-build verification OR release-pipeline-integrated
  regeneration.

### Phase 3 ŌĆö Test-quality

- **G1 mutmut**: `[tool.mutmut]` config in root `pyproject.toml`
  targets `gap_analyzer` + `risk_statements` modules. New
  `.github/workflows/mutmut.yml` runs weekly Sunday + on
  workflow_dispatch. Operator runbook in
  `docs/mutation-testing.md`.
- **G2 hypothesis**: 8 new property-based tests in
  `tests/property/` covering normalizer + crosswalk invariants.
  Configurable `ci` (default, derandomized, 200ms deadline) +
  `dev` (random, 1000ms deadline) profiles via
  `tests/property/conftest.py`.

### Phase 4 ŌĆö AI-quality

- **DFAH faithfulness scoring (P3.1)** ŌĆö second arXiv 2601.15322
  metric. New `evidentia_ai.eval.faithfulness` module with
  `FaithfulnessResult` Pydantic model + `faithfulness_score()`
  function using stdlib Jaccard token-overlap (no heavy ML
  deps). Default threshold 0.3 (conservative; catches gross
  hallucinations, misses paraphrases). v0.8.3 sentence-
  transformers semantic-similarity path planned. Operator guide
  in `docs/dfah-faithfulness.md`.
- **First-class Sigstore signing for `evidentia eval` output
  (P3.2)** ŌĆö `evidentia_ai.eval.signing` wraps the v0.7.x OSCAL
  Sigstore helpers. CLI: `--sign / --no-sign` flag on
  `stub-smoke` + `risk-determinism` (tri-state default
  auto-detects via `GITHUB_ACTIONS` env); new `evidentia eval
  verify` subcommand. New `EventAction.AI_EVAL_OUTPUT_SIGNED`
  audit entry.

### Phase 5 ŌĆö Pre-release-review v4 + ship

Full 7-step Pre-tag review per the v4 SKILL.md (Continuous-style
compression for the well-bounded diff). 5th canonical
deliverable [`docs/security-review-v0.8.2.md`](security-review-v0.8.2.md)
captures findings + 6-framework compliance mapping (NIST SSDF,
SLSA, ISO 27001, SOC 2, DORA, OpenSSF Scorecard, CISA Pledge).

## Re-execution recipe

```bash
# Verify the tagged release
uvx pypi-attestations verify pypi --repository \
  https://github.com/polycentric-labs/evidentia \
  "pypi:evidentia-0.8.2-py3-none-any.whl"
cosign verify ghcr.io/polycentric-labs/evidentia:v0.8.2 \
  --certificate-identity-regexp='^https://github.com/Polycentric-Labs/evidentia/' \
  --certificate-oidc-issuer=https://token.actions.githubusercontent.com
gh release download v0.8.2 -p '*sbom*' \
  && osv-scanner --sbom evidentia-sbom.cdx.json
docker run --rm ghcr.io/polycentric-labs/evidentia:v0.8.2 doctor
gh release view v0.8.2 --json body --jq '.body | length'  # expect Ōēź 1500

# Run the new gates locally
uv run pytest tests/property/                  # G2 hypothesis
uv run mutmut run                              # G1 mutmut (slow; 30-60 min)
evidentia mcp serve --allow-root /opt/data &   # F-V81-S1 path-gating
evidentia eval stub-smoke -o eval.json --sign  # P3.2 signing (CI w/ OIDC)
evidentia eval verify eval.json                # P3.2 verify
```
