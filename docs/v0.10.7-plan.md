## 1. Theme + scope rationale

v0.10.6 shipped clean (19th consecutive PROCEED-CLEAN), but its
post-cycle reviews surfaced ~9 code-quality items and the Step 7.D
verification surfaced 2 Scorecard alerts (#123 + #124) that explicitly
defer to v0.10.7. Layered on top: the Phase 2 (2026-05-27 continuation)
synthesis identified a 3-layer pre-push gate as the right structural
defense against the "ship-a-new-workflow-that-itself-trips-Scorecard"
pattern that produced #124.

**Strategic constraint** — Q3 2026 regulatory window:

| Event | Date | Days from 2026-05-27 |
|---|---|---|
| EU CRA reporting obligations live | 2026-09-11 | 107 |
| FedRAMP OSCAL mandate live | 2026-09-30 | 126 |
| Maryland MODPA (strictest US privacy law) | 2026-10-01 | 127 |
| CMMC Phase 2 Level 2 C3PAO mandatory | 2026-11-10 | 167 |

v0.11 federal-compliance theme (~6-8 weeks substantive) needs to land
**before** the CMMC Phase 2 trigger ideally — that's the
highest-leverage regulatory event for Evidentia's federal positioning.
Working backward: v0.11 must open no later than ~2026-08-15 to give a
6-week build cycle + 2-week buffer before 2026-11-10. v0.10.7 has a
2.5-month window before that backstop; the chosen 2-3-week scope
preserves ~2 months of v0.11 budget.

**Scope choice locked by Allen** (2026-05-27):

- Aggressive scope: Group A (code-quality polish) + Group D (Scorecard
  delta) + Pre-push Gate L2 (blocking layer only).
- Crosswalk hand-verification: **deferred** to v0.10.8 or v0.11 (the
  v0.10.6 `"self-attested-via-upstream"` disclaimer convention is honest).
- Pre-push Gate L1 (local Scorecard sweep) + L3 (warning-only): **deferred**.
  Rationale: L1 duplicates the scheduled CI Scorecard workflow with no
  new signal; L3's actionlint + online-pinact catches syntax errors
  that the GitHub Actions UI already surfaces. Marginal value of L1+L3
  doesn't justify ~1-2 extra weeks of v0.11 burn.

  third-party pin drift in `uv.lock` (closes the v0.10.0 F-V100-M1

  Inspect the diff for any non-workspace package whose version moved.

  fetch+auth step. The hook runs `--check` only when a
  `mappings/osps-baseline_*.json` or `_osps_upstream.py` file is in the
  push range (skip otherwise for speed). The unit-level drift contract

  (`.local/hooks/pre-push-bypass.log` JSONL — already gitignored
  via `.local/`).
- **D5.9** — Bypass logging: each bypass writes a JSONL row to
