## Theme

**"Composite action hardening + sample-data expansion."**

v0.7.0 brought the project to enterprise-grade. v0.7.1 closed the AI
features carry-over. v0.7.2 added supply-chain visibility (Scorecard)
+ contributor experience (IDE config) + hardening (catalog-drift
detector + IDE polish). v0.7.3 finishes the original v0.7.1-plan P1
supply-chain items that didn't make v0.7.2 (S1 SHA-pinning + S2 action
E2E smoke + S3 SLSA L3) plus opportunistic documentation polish from
the v0.7.2 review.

This is a small, focused release. Realistic ship target: 2-4 weeks
after v0.7.2.

## P0 — Composite action hardening (carries the v0.7.2 deferrals)

| # | Item | Scope | Rationale |
|---|---|---|---|
| S1 | **Composite action SHA-pinning** *(LANDED 2026-04-29)* | Replaced `@v5`, `@v4`, `@v2` with `@<40-char-sha> # vX.Y.Z` for all third-party actions in `.github/actions/gap-analysis/action.yml`. Scope **expanded** at land-time to also cover every `.github/workflows/*.yml` file (28 SHA-pinned refs across 6 files) so the Scorecard score reflects end-to-end pinned deps, not just the externally-consumed composite action. The single remaining tag-pinned `uses:` ref is `pypa/gh-action-pypi-publish@release/v1` — the documented PyPA pattern for trusted-publisher OIDC publishes. Dependabot's `github-actions` ecosystem (added in v0.7.2 post-audit hardening) opens grouped weekly PRs to advance the pins. | OpenSSF Scorecard "Pinned-Dependencies" check. |
| S2 | **Composite action E2E smoke test** *(LANDED 2026-04-29)* | `.github/workflows/action-smoke-test.yml` runs the `./.github/actions/gap-analysis` against the Meridian fintech v2 sample on every PR that touches the action surface, the CLI, or the sample data. Pre-installs `evidentia-core` editable from the PR's local source so the action runs against same-PR CLI (not latest PyPI). Asserts `gap-report.json` and `oscal-ar.json` land with the expected structure. Sigstore off (covered by S3); `fail-on-regression: false` because Meridian is a demo scenario. | Catches future CLI/action drift like the `--bundle` vs `--sigstore-bundle` mismatch surfaced in v0.7.0 Step 4. |
| S3 | **SLSA L3 build provenance** *(LANDED 2026-04-29)* | Added `actions/attest-build-provenance@v2.4.0` (SHA-pinned) to `.github/workflows/release.yml` between the SBOM-generation step and the PyPI-publish step so a single attestation covers all 6 wheels + 6 sdists + the CycloneDX SBOM. Added `attestations: write` to the job permissions. Verifiable via `gh attestation verify dist/<wheel> -R allenfbyrd/evidentia`. Closes the H2 enterprise-grade item ("SLSA L2+ reproducible builds + SBOM"); restores `gh attestation verify` as a working verifier alongside the existing `pypi-attestations verify pypi` path. The first SLSA-attested release will be v0.7.3 itself. |
