## Theme

**Concrete cloud-WORM backends + FAIR Monte Carlo simulation +
remaining P3 deferral closures.** Carries the v0.7.11 P0 audit

## P0 — Concrete WORM backends

The v0.7.11 P0 ships the abstract `WORMBackend` ABC + a
`LocalFilesystemWORM` reference impl. v0.7.12 ships the three
canonical cloud-backed implementations:

- **`S3ObjectLockWORM`** (extra: `evidentia[worm-s3]`)
  - boto3 + S3 Object Lock API
  - Compliance + governance modes
  - Legal-hold via `ObjectLockLegalHoldStatus`
  - Retention via `ObjectLockRetainUntilDate`
  - Tests via moto S3 mock (already in dev deps)
  - Estimated: 4-5 days

- **`AzureImmutableBlobWORM`** (extra: `evidentia[worm-azure]`)
  - azure-storage-blob + immutable blob policies
  - Time-based retention + legal-hold
  - Tests via azure-storage-blob test stubs
  - Estimated: 4-5 days

- **`GCSBucketLockWORM`** (extra: `evidentia[worm-gcs]`)
  - google-cloud-storage + bucket retention policy + lock
  - Tests via google-cloud-storage test fixtures
  - Estimated: 3-4 days

Total: ~2 weeks for all 3 backends + integration tests.

## P1.5 G4.1 — FAIR Monte Carlo simulation

The v0.7.11 P1.5 G4 ships deterministic PERT-mean expected-value
quantification. v0.7.12 adds the canonical FAIR simulation form:

- New `evidentia risk quantify --method fair-mc --iterations 10000`
- Beta-PERT distribution sampling for each PERT-range factor
- Per-iteration ALE + percentile distribution (P10 / P50 / P90)
- Box-and-whisker Markdown rendering
- Optional CSV export of raw simulation outputs
- Reuses `OpenFAIRScenario` schema unchanged (the deterministic
  + Monte Carlo paths share the input)

Estimated: 4-5 days (numpy.random.beta-PERT sampling + result
aggregation + report rendering).
