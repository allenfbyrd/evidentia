New directory `packages/evidentia-core/src/evidentia_core/crosswalks/data/` (matches the existing catalog-data convention `packages/.../catalogs/data/`). Five YAML files (was JSON in the original draft; revised 2026-05-26 to match catalog pattern + improve human review):

- `osps-baseline_to_eu-cra.yaml`
- `osps-baseline_to_pci-dss-4.0.yaml`
- `osps-baseline_to_nist-800-161.yaml`

Each file structure (inline `schema_version` instead of `$schema` URL — avoids the DNS-hosted-schema commitment per the 2026-05-26 review):

```yaml
schema_version: 1
provenance: upstream-osps-guidelines
verification: self-attested-via-upstream
verification_note: |
  Mappings auto-extracted from the OpenSSF OSPS Baseline `guidelines[]`
  array at upstream commit <sha>. Not independently verified against
  the target framework. Consumers requiring independent verification
  should plan a hand-check pass.
source_framework: osps-baseline-v2026.02.19
target_framework: nist-ssdf-800-218  # one of: nist-ssdf-800-218, nist-csf-2.0, eu-cra, pci-dss-4.0, nist-800-161
mappings:
  - source_control: OSPS-AC-01
    target_controls:
      - PO.1.1
      - PO.1.2
    upstream_reference: https://github.com/ossf/security-baseline/blob/<sha>/baseline/OSPS-AC-01.yaml#L42
```

Single mapping schema; consumers can hand-verify later by changing `verification: self-attested-via-upstream` → `verification: hand-checked-2026-MM-DD`. The v0.11 EU AI Act ↔ ISO 42001 crosswalk (currently planned at `docs/crosswalks/eu-ai-act-to-iso-42001.yaml` per integration-plan §11.5) **will move to this same `packages/.../crosswalks/data/` location at v0.11 cycle entry** — the integration-plan path expectation will be updated then; no v0.10.6 commit handles that.

## 6. Error handling

- **C1 catalog load test**: existing `tests/unit/test_catalogs/test_all_bundled.py` smoke suite auto-extends (parametrized) — new YAMLs must pass the smoke or C1 reverts.
- **C1 OSCAL validation**: pipeline runs `oscal-cli validate` (or Trestle) against `osps-baseline.oscal.json` — must pass or C1 reverts. (Tool availability check at execution time; if `oscal-cli` isn't on PATH, install via the Trestle pinned version and document in commit body.)
- **C3 CI gate failure modes**: any cited evidence link 404 → workflow fails; the conformance doc is intentionally fragile-to-link-rot so it cannot drift silently.
- **C5 crosswalk schema**: a unit test loads all 5 YAMLs against the inline `schema_version: 1` shape; any mapping row missing `source_control` or `target_controls` fails.
- **C6 collector retry/backoff**: reuses existing `evidentia-collectors.github` retry+backoff + cursor-pagination patterns from v0.10.5 idempotency hardening.
- **C7 workflow permissions audit**: writes a report to stdout; doesn't fail the build (advisory only this cycle; v0.10.7 can add a CI gate per §2.E).
- **GHSA enablement (T1, inside C2)**: idempotent — re-applying the PATCH is safe if it's already enabled. Pre-flight `gh api repos/.../security-advisories` returns-empty check captured in commit body.
- **OSCAL upstream PR (T2)**: separate publishing action; v0.10.6 ships independent of PR-merge timing.
