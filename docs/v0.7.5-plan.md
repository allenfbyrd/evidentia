## Theme

**"Container publish + remaining audit-cleanup carry-forward."**

v0.7.3 closed the OpenSSF Scorecard "Pinned-Dependencies" check
end-to-end, landed SLSA L3 build provenance, shipped the v0.8.0
forward plan, and added a non-publishing container-image build
smoke test (which had three wrong CLI invocations corrected by
the v0.7.4 hot-fix). v0.7.5 is the small, focused follow-up
release that:

1. Flips the container-image build from "smoke-test only" to
   "tag-triggered publish to `ghcr.io/allenfbyrd/evidentia` with
   cosign keyless OIDC signing"
2. Runs the quarterly `docs/positioning-and-value.md` re-sync
   (the cadence target was Q3 2026 — by the time v0.7.5 is in
   flight, the 3-month cadence date has either arrived or will
   imminently)
3. Fixes a pre-existing UX wart in `evidentia oscal verify`
   surfaced during the v0.7.3 capability matrix
4. Carries forward the optional / community items from
   [v0.7.3-plan.md](v0.7.3-plan.md) §P2

This is a small release. Realistic ship target: 4–6 weeks after
v0.7.3.

| R2 | **`evidentia oscal verify` UX fix when AR has no embedded evidence + no signature.** Currently returns `FAIL` with "No embedded evidence resources (digests skipped). No GPG signature checked. No Sigstore signature checked." That's misleading: the verifier ran 3 checks, all skipped because there was nothing to verify and no `--require-signature` was supplied — the right verdict is `PASS (no verification surface)` or similar. The exit code should reflect: 0 if no signature was required and digests-skipped is the only finding; 1 if `--require-signature` was set and no signature was found. **Surfaced during v0.7.3 Step 4 capability matrix; pre-existing — not introduced by v0.7.3.** | ~1 day. Touches `evidentia_core.oscal.verify.verify_ar_file` + the CLI's `_render_rich` / `_emit_json` exit-code logic. Add a regression test against an evidence-free AR. |

## P0.5 — Critical-security batch (S1-S6) — **NEW 2026-04-29**

GitHub code-scanning surfaced 37 open alerts during the v0.7.4
post-tag review. The 15 HIGH-severity findings represent real
attack surface in the FastAPI router code that must be closed
before the v0.8.0 AI moat work adds more surface. This bucket is
the v0.7.5 P0.5 — sits between P0 (container publish) and P1
(quarterly cadence) in priority because it's pre-existing security
debt, not new work.

| # | Item | Scope | Severity |
|---|---|---|---|
| **S1** | **Path-injection containment helper** + refactor of all 14 callsites | New `evidentia_core.security.paths.validate_within(path, safe_root)` helper that resolves the path and checks `is_relative_to(safe_root)`. Refactor the 14 alert-flagged callsites in `evidentia_api/routers/{risks,integrations,gaps}.py`, `evidentia_api/app.py`, and `evidentia_core/gap_analyzer/inventory.py` to use it. Add unit tests covering symlink-traversal + URL-encoded `..` + absolute-path-injection inputs. | 14 HIGH |
| **S2** | **ReDoS remediation** in `evidentia_core/models/catalog.py:42` | Replace the polynomial-time alternation with a bounded character class + cap input length at the model-validation boundary. Add a regression test with a known-pathological input. | 1 HIGH |
| **S3** | **API stack-trace exposure** fix in `evidentia_api/routers/integrations.py:61, 66, 72` | Catch + log internally via the existing `evidentia_core.audit.logger`; return generic 500 messages externally. Audit the rest of the router code for the same pattern (Allen's grep should find any remaining instances). | 3 MEDIUM |
| **S4** | **Workflow permissions hygiene** in `.github/workflows/test.yml:47, 71, 112` | Add `permissions: contents: read` to the test, lint, and typecheck jobs. The other workflows already declare permissions explicitly; test.yml was overlooked. ~10 min. | 4 MEDIUM |
| **S5** | **Pinned-Dependencies triage** | (a) Document floating `apt-get install -y` package versions in `Dockerfile` lines 25 + 59 with a comment justifying the floating intent (security patches + base-image-rebuild cadence). (b) Add a Scorecard-suppression comment to `action-smoke-test.yml:63` for the intentional `pip install -e packages/evidentia-core` line. | 5 MEDIUM |
| **S6** | **URL-substring-sanitization review** in `tests/unit/test_network_guard.py:133, 178` | Test code only — not a runtime vuln. Review the `network_guard` module for the same `==` vs hostname-prefix-match pattern; refactor to use parsed URL hostname comparison. ~1 hour. | 2 HIGH (test-only) |

**Rationale for v0.7.5 placement**: closing 15 HIGH findings before
shipping more public surface (the v0.7.7 SQL collectors + v0.7.8
cloud-DW collectors will substantially expand the API + library
attack surface). Better to fix path-injection NOW than discover it
applies to 5 new SQL connection-string code paths.

three wrong CLI invocations (`evidentia --version` →
`evidentia version`; `evidentia frameworks list` →

work + a pre-existing latent same-pattern bug in the composite
action's install step. The originally-planned v0.7.4 scope is
unchanged; only the version number moved.

The pointer in MEMORY.md is updated to reference this plan;
v0.7.3 + v0.7.4 SHIPPED + v0.7.5 NEXT bookkeeping happens at
v0.7.4 tag-push time.
