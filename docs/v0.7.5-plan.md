## Theme

**"Container publish + remaining audit-cleanup carry-forward."**

v0.7.3 closed the OpenSSF Scorecard "Pinned-Dependencies" check
end-to-end, landed SLSA L3 build provenance, shipped the v0.8.0
forward plan, and added a non-publishing container-image build
smoke test (which had three wrong CLI invocations corrected by
the v0.7.4 hot-fix). v0.7.5 is the small, focused follow-up
release that:

1. Flips the container-image build from "smoke-test only" to
   "tag-triggered publish to `ghcr.io/allenfbyrd/evidentia` with
   cosign keyless OIDC signing"
2. Runs the quarterly `docs/positioning-and-value.md` re-sync
   (the cadence target was Q3 2026 — by the time v0.7.5 is in
   flight, the 3-month cadence date has either arrived or will
   imminently)
3. Fixes a pre-existing UX wart in `evidentia oscal verify`
   surfaced during the v0.7.3 capability matrix
4. Carries forward the optional / community items from
   [v0.7.3-plan.md](v0.7.3-plan.md) §P2

This is a small release. Realistic ship target: 4–6 weeks after
v0.7.3.

## P0 — Container publish + cosign signing

| # | Item | Scope |
|---|---|---|
| C1 | **`release-container.yml` workflow** — tag-triggered companion to the existing `release.yml` PyPI publish path. Builds the repo-root `Dockerfile` via `docker/build-push-action`, pushes to `ghcr.io/allenfbyrd/evidentia:v<X.Y.Z>` AND `ghcr.io/allenfbyrd/evidentia:latest`, attaches the resulting image digest to the GitHub Release alongside the wheels + SBOM. | New workflow file. Permissions: `packages: write` (ghcr push) + `id-token: write` (cosign keyless OIDC) + `contents: write` (release attach). Mirrors the SLSA L3 build-provenance pattern from `release.yml`. |
| C2 | **cosign keyless signing** on every published image tag — same OIDC-bound identity as the PyPI Trusted Publisher path. Verify via `cosign verify --certificate-identity-regexp '...allenfbyrd/evidentia/...' --certificate-oidc-issuer 'https://token.actions.githubusercontent.com' ghcr.io/allenfbyrd/evidentia:v<X.Y.Z>`. | Signing call lives in `release-container.yml`. Documentation update in `docs/sigstore-quickstart.md` to extend the Sigstore narrative to container artifacts. |
| C3 | **SLSA build provenance for the image** via `actions/attest-build-provenance@v2` extended to attest the image digest (in addition to the wheel + sdist + SBOM coverage already in `release.yml`). | Either inline in `release-container.yml` or as a shared composite step in `release.yml` that the new workflow calls. Closes the container-image half of enterprise-grade L1. |
| C4 | **Update enterprise-grade.md L1** from "Repo-root Dockerfile + CI smoke-test, not yet published" → "Published to ghcr.io with cosign signing + SLSA build provenance per release." | One-line status flip + cross-link refresh. |

## P1 — Quarterly research-resync + UX fix

| # | Item | Scope |
|---|---|---|
| R1 | **`docs/positioning-and-value.md` quarterly re-sync** (DOC5 carry-forward from v0.7.3) — Q3 2026 cadence target (~July 2026). Per the [release-checklist](release-checklist.md) §"Step 11 — Quarterly cadence". Snapshot the current version to `docs/positioning-and-value-2026-Q3.md`, re-run the 7 research streams (Vanta-class commercial GRC; OSS ecosystem; industry signals + regulatory + M&A; academic + scholarly; AI/LLM tools; industry voices; internal capability inventory), semantically diff against the prior version, promote the new synthesis if material change. Document the diff materiality in `docs/positioning-and-value.md` §16 Version history. | ~3-6 hours of parallel agent research per the existing doc's 7-stream recipe. |
| R2 | **`evidentia oscal verify` UX fix when AR has no embedded evidence + no signature.** Currently returns `FAIL` with "No embedded evidence resources (digests skipped). No GPG signature checked. No Sigstore signature checked." That's misleading: the verifier ran 3 checks, all skipped because there was nothing to verify and no `--require-signature` was supplied — the right verdict is `PASS (no verification surface)` or similar. The exit code should reflect: 0 if no signature was required and digests-skipped is the only finding; 1 if `--require-signature` was set and no signature was found. **Surfaced during v0.7.3 Step 4 capability matrix; pre-existing — not introduced by v0.7.3.** | ~1 day. Touches `evidentia_core.oscal.verify.verify_ar_file` + the CLI's `_render_rich` / `_emit_json` exit-code logic. Add a regression test against an evidence-free AR. |

## Acceptance criteria

v0.7.5 is "shipped" when **all** of these pass:

- [ ] `release-container.yml` exists and is green on the v0.7.5 tag push (C1)
- [ ] `ghcr.io/allenfbyrd/evidentia:v0.7.5` is published, cosign-signed, and verifiable via `cosign verify` (C2)
- [ ] `actions/attest-build-provenance` covers the image digest (C3); `gh attestation verify` resolves against the published image
- [ ] `docs/enterprise-grade.md` L1 status flipped (C4)
- [ ] `docs/positioning-and-value.md` re-synced; diff documented in §16 Version history (R1)
- [ ] `evidentia oscal verify` UX fix landed; regression test passes (R2)
- [ ] All v0.7.x baselines hold: pytest 965+ passing, mypy strict clean, ruff lint clean
- [ ] `pre-release-review` skill (Pre-tag variant) all 6 steps green
- [ ] `pypi-attestations verify pypi` green on all 6 published wheels
- [ ] `gh attestation verify` green on the SLSA build-provenance attestation (continuing v0.7.3+'s pattern)
